• Class Timetable网页APP

    灵感来源

    每当新学期公布课表时，我们常常只能在E-bridge上把课表截图存在相册中, 然后不得不经常反复的去相册翻看自己的课表来知道下一节课是什么，有些同学还会把截图设置为手机壁纸，总之美观和便捷不能兼顾。在过去的这两年，我一直使用着iStudiez Pro来手动添加课程，并用日历管理着自己的课程，虽然用户体验很不错，但是手动添加的过程是痛苦的而且不能准确的跳过间隔周、节假日以及考试周。所以在假期我就萌生了一个想法，希望能通过程序来抓取ebridge上的课表并智能的满足以上所有需求，最重要的是，能一键添加至系统的日历中。

    使用技术的简单介绍

    • Python

      作为目前增长最快的主流语言，Python对于人工智能、大数据、深度学习有着非常重要的作用，无论你是不是程序员，学习Python都是非常有用的。在这次项目中，Python不仅负责了他最拿手的爬虫，课表的解析和日历的生成也由Python完成。

    • PHP

      PHP负责网站的后端，尽管其高并发的应对能力不如Node.js，但PHP的易用性和高拓展性以及丰富的支持文档使得它成为世界上最好的编程语言 : )

    • HTML, CSS, JavaScript

      网页前端三剑客负责网页页面的外观显示部分，引入的Bootstrap, jQuery, FontAwesome可以高效便捷的优化页面的显示以及不同设备自适应。

    学习、编程过程

    • 学习

      在这里想说的一点就是：学一门语言或技术，想获取一个系统的知识体系的想法固然是对的，看教辅或API文档也是必不可少的，但是一直停留在理论阶段不仅效率较低而且容易失去兴趣。实践永远是步最快的方法，跟着项目去做，哪里不会搜哪里，每遇到一个问题，去搜索引擎寻找答案，这样你不仅学会了也会对遇到的问题有深刻的印象，而当你有一定的知识积累后再去看系统的讲解，也会更加顺畅。

    • 项目编程过程

      这次项目的整体逻辑过程是这样的：

      1. 通过HTML,CSS,Javascript搭建出用户界面。具体包括：设计网页整体UI、设计表单提供用户输入用户名密码的入口、设计并撰写用户说明文档以及相关支持（产品开发必不可少的一步）

      2. 利用Python编写爬虫模拟登录E-bridge。在制作爬虫的过程中要注意的是：在当今大多数网站中，通过提交表单来登录网站中的表单不仅包含了用户名和密码，仔细观察登录页面的HTML代码可以发现，表单中还包含了流水单号、时间戳、用户身份识别符等信息，提交表单时也要记得将这些信息一并提交。

      3. 访问到课表页面的HTML代码后对其的解析就是本次项目的核心部分了。首先最重要的就是需要了解并熟悉网页代码的结构，在这里强烈推荐电脑端Chrome浏览器的元素审查器，可以通过鼠标选取可视区域的元素来快速查看元素之间的嵌套结构以及代码。了解课表的代码结构后就可以相应的去解析、分门别类的筛选归类，获取课程的所有相关信息。

      4. 日历文件生成。考虑到课程信息已经在上一步存储在Python对象当中，日历文件的生成直接调取对象也更加方便，所以依旧由Python完成。iCalendar作为全球最广为支持的日历协议，所有平台设备都兼容这个协议，.ics格式被应用的最为广泛。所以按照ics文件的格式去添加课程事件的时间地点等参数，日历文件就制作好了。

      5. 文件的制作搞定后，那就剩下提供用户下载了。第一步做好了登录的表单，那么在这一步就要将前端和后端连接起来。首先用户输入的用户密码属于敏感信息，不能通过GET方式直接传递，而要用更加安全的POST方式传递给后端，后端PHP获取信息后调用之前写好的Python程序，执行爬虫、解析、日历生成的完整流程，然后后端去获取这个文件提供下载，并删除该文件。

      6. 最后就是一些完善工作，比如下载成功提示，提升用户体验。

    开发中的困难和解决方法

    • 由于E-bridge上的课表呈现方式不同于Excel或者MySQL，并不能通过直接选择列数（星期几）来获取当日的课表，而是需要按照HTML中table的格式进行解析，根据表格题头星期所在列数在表格主体行中进行推算，这样才能准确定位当天的每一节课。
    • 在一开始，程序雏形能很轻松的将我自己ICS的‘普通’课表提取出来，但是在测试电气系的多列课表时就完全乱套了，多列的课程会按照每天一节向后串位。这个时候就要考虑如何设计算法来相应完整的抓取所有列的内容。解决方式不止一种，虽然每行抓取完整的一行看似方便，但是不方便区分课程所在的天数，所以我采用的是逐列抓取，方便将课程归入其相应的天内。至于一天内多列课程的情况就需要程序算法检测当日的列数，这样才可以防止课程串入下一天。
    • 添加日程本身很简单，但是如何根据课表中指定的周数设置循环事件却很麻烦，这里我能想到的较为简陋方法就是在有断周的情况时新建一个相同的事件，并继续设置为每周循环一次。考试周节假日一并考虑为断周，这样问题被简单化后也就迎刃而解了。

    成品的功能和亮点介绍

    成品做好后也算是自己制作的第一个被大家认可的实用小工具了，同学们可以一次性把自己所有的课程添加至日历，而操作过程仅仅是登录验证，然后就可以轻松的通过日历应用快速的查看管理自己接下来的课程，如果你有智能手表，一抬手就可以知道自己下一堂课是什么，通知中心也一样很方便，甚至可以给课程加上闹钟提醒。

    亮点嘛，就不赘述太多了：

    • 成品可以精确的获取显示在ebridge中个人课表页面的所有课程，尤其是对电气系一天内多达6-7列的课表进行了优化。

    • 在添加至日历文件的过程中，程序可以准确的按照课程所在周数进行添加日程，能识别各种不规律的课程频率。此外程序设置日历项目避开了十一假期以及期中考试周，让这份课程表真正成为用户日程表的一部分。

    总结这次收获的经验

    在做这次项目之前我也做过几个网页的项目，但是这是我第一次独立的从想法一点点做成成品，不仅了解了爬虫， Python，ics日历的一些知识技能，而且了解了一些类似网络工具提供服务的大致原理。而且为了提供给同学下载使用，也仔细思考了用户的使用流程，测试了不同情况可能出现的问题，甚至在优化服务器的高并发多线程能力进行了一些尝试。
